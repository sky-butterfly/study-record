# 용어정리

1. SCM : Software Configuration Management, 형상관리 툴
2. CI : Continuous Integration, 지속적 통합 툴 (ex. jenkins)
3. 서드파티 (3rd Party) : 제작사와 사용자를 이어주는 중간다리로, 개인 개발자나 프로젝트 팀, 혹은 외부 업체에서 개발한 플러그인, 라이브러리, 프레임워크 등을 서드파티로 볼 수 있다. 
4. 프로비저닝 : 사용자의 요구에 맞게 시스템 자원을 할당, 배치, 배포해 두었다가 필요 시 시스템을 즉시 사용할 수 있는 상태로 미리 준비해 두는 것
5. Endpoint : 메소드 같은 URI에 대해서 다른 요청을 하도록 구별해주는 항목 (ex. /books, /dogs)
