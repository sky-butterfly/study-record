## RPC
- 참고 : https://real-dongsoo7.tistory.com/131

### RPC의 개념
- Remote Procedure call
- 별도의 원격 제어를 위한 코딩 없이 다른 주소 공간에서 함수 또는 프로시저를 실행할 수 있게 해주는 프로세스간 통신
- RPC를 통해 위치에 상관없이 원하는 함수 또는 프로시저 사용이 가능
- IPC(inter-Process Communication : 프로세스간 통신) 방법 중 한 종류
- MSA 환경에서 서비스간에 많이 사용되는 방식

### 동작과정
- RPC는 클라이언트/서버 구조를 기반으로 작동

1. IDL(Interface Definition Language)를 통해 호출 인터페이스 정의
2. rcpgen(유틸리티)를 통해 클라이언트의 stub와 서버의 skeleton 생성
3. 클라이언트는 리모트 프로시저를 사용하기 위해 설계된 스텁의 프로시저를 호출하며, 프로시저 호출에 필요한 인자와 비지니스 로직에 필요한 메소드 호출
4. 스텁은 서버가 이해할 수 있는 형태로 데이터 캐스팅 진행하고, 서버 측 RPC로 호출
5. 서버는 수신된 호출에 대한 데이터 처리
6. 서버 RPC는 처리된 데이터를 캐스팅하여 클라이언트로 응답
